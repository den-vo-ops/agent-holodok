# Дизайн: главное меню бота на reply-клавиатуре

> Дата: 2026-07-04
> Основа: spec.md §6 (UX), §5 (функциональный скоуп), решения владельца в чате.
> Статус кодовой базы на момент дизайна: Фазы 1A+1B в `master`; разведка/аналитика (1C/1D) ещё не реализованы.

## 1. Цель

Добавить постоянное главное меню на **reply-клавиатуре** (кнопки снизу под полем ввода), чтобы нетехнический мастер видел основные функции агента сразу, а не только inline-меню контента после `/start`.

## 2. Раскладка

Постоянная reply-клавиатура (`resize_keyboard=True`, `is_persistent=True`, не скрывается после нажатия):

```
[ ✍️ Создать контент ] [ 📊 Показать отчёт ]
[ 🔍 Спросить о рынке ] [ 📝 Мои правила ]
[ ⚙️ Обучить стиль ]
```

Пять кнопок (решение владельца — полный набор сразу, за неготовыми функциями заглушки).

## 3. Поведение кнопок

| Кнопка | Поведение | Статус |
|---|---|---|
| ✍️ Создать контент | открывает существующее inline-меню сценариев (`build_scenario_menu`) | живая |
| 📝 Мои правила | показывает список сохранённых жёстких правил (`db.get_hard_rules`) + подсказку «чтобы добавить, напиши: запомни, …». Пустой список — отдельное сообщение | живая |
| ⚙️ Обучить стиль | перезапускает онбординг: переводит в состояние `Onboarding.waiting_for_samples`, по `/done` перезаписывает профиль стиля новыми образцами | живая |
| 📊 Показать отчёт | заглушка: «Отчёты появятся после подключения разведки (Фаза 1C/1D)» | заглушка |
| 🔍 Спросить о рынке | заглушка: тот же текст-заглушка | заглушка |

## 4. Изменения по файлам

| Файл | Изменение |
|---|---|
| `holodok_agent/bot/keyboards.py` | новая чистая функция `build_main_menu() -> ReplyKeyboardMarkup` + константа с 5 подписями кнопок. Импорт `ReplyKeyboardMarkup`, `KeyboardButton` |
| `holodok_agent/bot/handlers.py` | `handle_start` при готовом профиле показывает главное меню (текст-приветствие + `build_main_menu()`), а не сразу inline-сценарии. Новые хендлеры: `handle_menu_create_content`, `handle_menu_my_rules`, `handle_menu_retrain_style`, `handle_menu_stub` (общий для двух заглушек). Константы подписей и текста-заглушки |
| `holodok_agent/db.py` | без изменений (`get_hard_rules` уже существует) |

## 5. Ключевые технические решения

1. **Распознавание нажатий.** Reply-кнопки шлют обычное текстовое сообщение с текстом кнопки (не callback). Хендлеры регистрируются фильтром точного совпадения `F.text == <подпись>`. Подписи выносятся в именованные константы, используемые и в клавиатуре, и в фильтрах — единый источник правды.
2. **Порядок регистрации в роутере.** Хендлеры кнопок регистрируются **до** `handle_remember_rule` и до `handle_scenario_input`/онбординг-хендлеров, чтобы текст кнопки не был перехвачен правилом «запомни…» или сборщиком образцов. Кнопки-фильтры точного совпадения — самые специфичные, идут первыми.
3. **Inline-меню сценариев сохраняется без изменений** — просто теперь вызывается по кнопке «Создать контент», а не автоматически после `/start`.
4. **Заглушки** — общий хендлер, отвечающий строковой константой `STUB_MESSAGE`. При реализации Фаз 1C/1D заглушка заменяется реальным вызовом.
5. **Онбординг из кнопки** переиспользует существующую машину состояний `Onboarding` и `handle_onboarding_done` — новой логики анализа стиля не добавляется, только точка входа.

## 6. Тестирование (TDD, CLAUDE.md §6)

`tests/test_keyboards.py`:
- `build_main_menu()` возвращает `ReplyKeyboardMarkup` с 5 кнопками, подписи и раскладка (3 ряда: 2+2+1) соответствуют ожидаемым.

`tests/test_handlers.py`:
- нажатие «Создать контент» → ответ содержит inline-меню сценариев;
- «Мои правила» с непустым списком → ответ перечисляет правила; с пустым → сообщение-подсказка;
- «Обучить стиль» → состояние переведено в `Onboarding.waiting_for_samples`;
- нажатие заглушки → ответ равен `STUB_MESSAGE`;
- `handle_start` при готовом профиле → ответ содержит `build_main_menu()`.

Хендлеры тестируются прямым вызовом с `MagicMock`/`AsyncMock` (как остальные в проекте), без поднятия Dispatcher.

## 7. Вне скоупа

- Реальная логика отчётов и вопросов о рынке (Фазы 1C/1D) — только заглушки.
- Настройка Telegram Bot Menu (команды слева от поля ввода) — не делаем, выбран reply-вариант.
- Удаление правил через кнопку — только просмотр и подсказка по добавлению.
