# Дизайн: онбординг и справка

**Дата:** 2026-07-04
**Статус:** одобрен владельцем, готов к плану реализации

## Проблема

Новый человек (или сам владелец при первом запуске) не понимает, как бот
работает и как им пользоваться. Сейчас первый `/start` молча бросает в загрузку
образцов текста, нет объяснения возможностей, нет команды справки, нет способа
повторно прочитать инструкцию. Кнопки-заглушки дают сухое техническое сообщение.

## Цель

При первом запуске — понятный онбординг из 3 сообщений. Понятные объяснения при
нажатии на каждую кнопку. Команда «Помощь» для повторного знакомства с
инструкцией. Онбординг показывается только новым пользователям.

## Поведение

1. **Первый `/start`** (пользователь не отмечен как «уже видел онбординг»):
   отправить 3 сообщения онбординга → отметить пользователя как onboarded →
   показать главное меню. НЕ входить в FSM загрузки образцов.
2. **Повторный `/start`** (уже onboarded): сразу главное меню, без 3 сообщений.
3. **Команда `/settov`**: запускает загрузку тон-оф-войс (состояние
   `Onboarding.waiting_for_samples`, тот же промпт «пришли объявления → /done»).
   Кнопка «⚙️ Обучить стиль» использует тот же обработчик.
4. **Команда `/help` и кнопка «❓ Помощь»**: показывают полную инструкцию с
   разбором каждой команды и кнопки. Оба сбрасывают активный FSM-стейт
   (консистентно с текущим поведением кнопок меню, которые прерывают поток).
5. **Мини-подсказка** перед действием каждой рабочей кнопки. Кнопки-заглушки
   («Показать отчёт», «Спросить о рынке») дают понятное «что это будет и когда»,
   каждая — свой текст.
6. Команды `/start`, `/settov`, `/help` регистрируются в меню команд Telegram
   через `bot.set_my_commands(...)` в `main.py`.
7. Сообщение-заглушка при попытке генерации без загруженного стиля меняется на
   «Сначала загрузи свой стиль — команда /settov (или кнопка ⚙️ Обучить стиль).»

## Определение «нового» пользователя

Нельзя переиспользовать «есть профиль стиля» — это другое понятие. Новая таблица:

```sql
CREATE TABLE IF NOT EXISTS onboarded_users (
    user_id INTEGER PRIMARY KEY,
    first_seen_at TEXT NOT NULL
);
```

Функции в `db.py`:
- `has_onboarded(conn, user_id) -> bool`
- `mark_onboarded(conn, user_id) -> None` — идемпотентна (`INSERT OR IGNORE`).

## Тексты

### Онбординг 1/3 — что умеет
> 👋 Привет! Я — твой личный помощник по контенту.
>
> Что я умею:
> ✍️ Пишу посты для ВК/Telegram, объявления для Авито/Юлы и ответы на отзывы — в твоём стиле.
> 📝 Запоминаю твои правила (например: «запомни, всегда указывай гарантию 1 год»).
> 📊 Готовлю аналитику по конкурентам и лидам (скоро).
>
> Главное — я подстраиваюсь под твою манеру писать, чтобы тексты звучали как твои, а не шаблонные.

### Онбординг 2/3 — как загрузить тон-оф-войс
> Чтобы я писал именно твоим голосом, покажи мне примеры своих текстов.
>
> Отправь команду /settov и пришли 3–5 своих старых объявлений или постов (по одному сообщению). Когда закончишь — напиши /done.
>
> Я разберу твой тон, лексику и структуру и буду держать их в каждом тексте. Захочешь обновить стиль позже — снова /settov или кнопка «⚙️ Обучить стиль».

### Онбординг 3/3 — быстрый старт
> 🚀 Быстрый старт:
> 1. /settov — загрузи свой стиль (3–5 текстов, потом /done).
> 2. Кнопка «✍️ Создать контент» — выбери пост / объявление / ответ на отзыв.
> 3. Проверь черновик → «Переделай» или «Опубликовал».
>
> Кнопки меню всегда внизу экрана. Забыл, что к чему? Жми «❓ Помощь» или /help.
>
> Погнали! 👇

*(после 3-го сообщения — главное меню)*

### /help / кнопка «❓ Помощь»
> ❓ Как пользоваться ботом
>
> Команды:
> • /settov — загрузить/обновить твой стиль (пришли 3–5 текстов, потом /done).
> • /help — показать эту инструкцию.
> • «запомни, …» — сохранить правило (напр. «запомни, всегда пиши телефон в конце»).
>
> Кнопки меню:
> ✍️ Создать контент — выбрать формат (пост ВК/Telegram, объявление Авито/Юла, ответ на отзыв, «дай идею») и получить черновик в твоём стиле.
> 📊 Показать отчёт — еженедельная сводка по конкурентам и лидам (появится в следующей фазе).
> 🔍 Спросить о рынке — быстрый вопрос про цены и спрос (появится в следующей фазе).
> 📝 Мои правила — список правил, которые я всегда учитываю.
> ⚙️ Обучить стиль — то же, что /settov.
> ❓ Помощь — эта инструкция.
>
> Под черновиком: «Переделай» — сгенерировать заново, «Опубликовал» — отметить, что выложил (для статистики).

### Мини-подсказки
- ✍️ Создать контент → «✍️ Соберу черновик в твоём стиле. Выбери формат:» + меню сценариев
- 📝 Мои правила → «📝 Это правила, которые я всегда держу в голове при генерации.» + список (или текст «пока нет правил»)
- 📊 Показать отчёт → «📊 Здесь будет еженедельная сводка по конкурентам и горячим лидам. Появится после подключения разведки и аналитики (Фаза 1C/1D).»
- 🔍 Спросить о рынке → «🔍 Здесь можно будет быстро спросить про цены и спрос по рынку Симферополя. Появится после подключения разведки (Фаза 1C/1D).»

## Тестирование (TDD)

- `db.py`: `has_onboarded` False до, True после `mark_onboarded`; повторный
  `mark_onboarded` не падает.
- `messages.py`: `ONBOARDING_MESSAGES` длиной 3; во 2-м сообщении есть `/settov`;
  `HELP_MESSAGE` упоминает `/settov`, `/help` и метки всех кнопок меню.
- `handlers.py`: новый пользователь получает 3 сообщения онбординга + меню, стейт
  НЕ `waiting_for_samples`, `mark_onboarded` вызван; известный пользователь —
  только меню; `handle_settov` ставит `waiting_for_samples`; `handle_help` шлёт
  `HELP_MESSAGE`; заглушки дают разные тексты.

## Вне скоупа

- Реальная функциональность отчётов и вопросов о рынке (Фаза 1C/1D).
- Мультипользовательский доступ (роутер по-прежнему фильтрует whitelist владельца).
